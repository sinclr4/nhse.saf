    Introduction #

    1. This paper details the services provided by the Technical Architecture Profession.
    2. This paper is aimed at the Technical Architecture Profession and its members to act as a guide for carrying out Technical Architecture work. A separate paper and information on the intranet exists to convey the Technical Architecture services to people outside of the Technical Architecture Profession.
    3. The Technical Architecture Profession covers the roles of Technical Architects, Solution Architects and Enterprise Architects. For the purpose of this paper, these are referred to as Technical Architect.
    4. In order to avoid overuse of the term Service, these services are referred to as a TASC Item. (Technical Architecture Service Catalogue Item).
    5. The deliverables for each offering detailed in this service catalogue should be considered the minimum for each engagement and are intended to promote the relationship with the Enterprise Architecture. The full deliverable set must be agreed with the customer prior to commencing any engagement.
    6. The TASC Items define the deliverables, which in turn are referenced in any work package definition.
    7. Each TASC Item is supported by accompanying guidelines and templates.
    8. The process for engagement of TASC Items can is for Programmes etc. to go through Resource Management with a request. If a programme is already working with a Technical Architect, the Technical Architect should help to create this request, otherwise, initial enquiry can be made through the Technical Architecture mailbox (technicalarchitecture@nhs.net).
    9. Manage improvements to this service catalogue can be achieved by contacting technicalarchitecture@nhs.net with the change.
